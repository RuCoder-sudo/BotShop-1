## 🎯 Описание проекта

SHOP-Q - это автоматизированный Telegram бот для создания маркетплейса виртуальных товаров. Бот предоставляет возможность множественным продавцам размещать свои товары, привязанные к конкретным географическим локациям (страна → город → район), а покупателям — удобно находить и покупать товары в своем регионе.

### ✨ Ключевые особенности:

- ✅ Многоуровневая система ролей: **Super Admin / Manager (Seller) / Buyer**
- ✅ Географическая иерархия для каждого товара (страна → город → район)
- ✅ Привязка товаров к локации
- ✅ Модульная структура (управление категориями, акции, реферальная система, техподдержка)
- ✅ Гибкие настройки в `config.py`
- ✅ Система верификации продавцов
- ✅ Корзина покупок и избранное
- ✅ Рейтинговая система (1.1 - 5.5 звезд)

### Описание ключевых файлов:

#### `main.py`
Главный файл приложения. Инициализирует бота, базу данных, регистрирует все обработчики команд и запускает polling.

**Основные функции:**
- `main()` - точка входа, создает Application и запускает бота
- `start_command()` - обработчик команды /start
- `role_selection_callback()` - обработка выбора роли пользователем

#### `config.py`
Файл конфигурации со всеми настройками бота.

**Основные параметры:**
- `BOT_TOKEN` - токен Telegram бота (из переменных окружения)
- `SUPER_ADMIN_IDS` - список Telegram ID администраторов
- `NOTIFICATION_CHANNEL_ID` - ID канала для уведомлений
- `PAYMENT_METHODS` - настройки платежных систем
- `REFERRAL_BONUS_*` - бонусы реферальной программы
- `ANTI_FLOOD_*` - настройки защиты от флуда

#### `database/database.py`
Класс для работы с SQLite базой данных. Содержит методы для:
- Управления пользователями
- Работы с локациями (страны, города, районы)
- Управления категориями
- Работы с товарами
- Обработки заказов
- Корзины и избранного
- Рейтингов и отзывов
- Верификации продавцов

#### `handlers/start.py`
Обработчики начального взаимодействия с ботом:
- Регистрация новых пользователей
- Выбор роли (покупатель/продавец)
- Отображение главного меню для каждой роли

#### `handlers/admin.py`
Все функции для администратора:
- Управление пользователями
- Верификация продавцов
- CRUD операции с локациями (создание, редактирование, удаление)
- CRUD операции с категориями
- Рассылки сообщений
- Статистика
- Инструменты и настройки

#### `handlers/manager.py`
Функционал для продавцов:
- Создание товаров (многошаговый процесс)
- Просмотр своих товаров
- История продаж
- Статистика
- Информация и поддержка

#### `handlers/buyer.py`
Функционал для покупателей:
- Выбор локации и просмотр магазинов
- Просмотр товаров
- Корзина покупок
- Избранное
- История заказов
- Профиль
- Поиск товаров

#### `utils/helpers.py`
Вспомогательные функции:
- `generate_referral_code()` - генерация реферального кода
- `generate_order_number()` - генерация номера заказа
- `check_admin()` - проверка прав администратора
- `check_manager()` - проверка роли продавца
- `format_price()` - форматирование цены
- `format_rating()` - форматирование рейтинга

## 🚀 Установка и настройка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd shop-q
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

**Зависимости:**
- `python-telegram-bot==20.8` - библиотека для работы с Telegram Bot API
- `python-dotenv==1.0.0` - загрузка переменных окружения из .env файла

### 3. Получение токена бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Получите токен в формате: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

### 4. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN=сюда токен бота
NOTIFICATION_CHANNEL_ID=сюда ID группы для уведомлений
SUPER_ADMIN_IDS=сюда ID админа
```

**Или** установите переменные окружения напрямую:

```bash
export BOT_TOKEN="сюда токен бота"
export NOTIFICATION_CHANNEL_ID="сюда ID группы для уведомлений"
export SUPER_ADMIN_IDS="сюда ID админа"
```

### 5. Настройка администраторов

Откройте `config.py` и укажите ваш Telegram ID в `SUPER_ADMIN_IDS`:

```python
SUPER_ADMIN_IDS = [сюда ID админа]
```

## 🖥️ Автозапуск на сервере

### Вариант 1: systemd (Linux)

Создайте файл сервиса `/etc/systemd/system/shop-q.service`:

```ini
[Unit]
Description=SHOP-Q Telegram Bot
After=network.target

[Service]
Type=simple
User=your_username
WorkingDirectory=/path/to/shop-q
Environment="BOT_TOKEN=сюда токен бота"
Environment="NOTIFICATION_CHANNEL_ID=сюда ID группы для уведомлений"
ExecStart=/usr/bin/python3 /path/to/shop-q/main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

**Активация:**

```bash
sudo systemctl daemon-reload
sudo systemctl enable shop-q
sudo systemctl start shop-q
sudo systemctl status shop-q
```

**Управление:**

```bash
sudo systemctl stop shop-q      # Остановка
sudo systemctl restart shop-q   # Перезапуск
sudo journalctl -u shop-q -f    # Просмотр логов
```

### Вариант 2: Docker

Создайте `Dockerfile`:

```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

ENV BOT_TOKEN=""
ENV NOTIFICATION_CHANNEL_ID=""

CMD ["python", "main.py"]
```

Создайте `docker-compose.yml`:

```yaml
version: '3.8'

services:
  shop-q-bot:
    build: .
    container_name: shop-q
    restart: unless-stopped
    environment:
      - BOT_TOKEN=${BOT_TOKEN}
      - NOTIFICATION_CHANNEL_ID=${NOTIFICATION_CHANNEL_ID}
    volumes:
      - ./shop_q.db:/app/shop_q.db
      - ./shop_q.log:/app/shop_q.log
      - ./images:/app/images
```

**Запуск:**

```bash
docker-compose up -d
docker-compose logs -f  # Просмотр логов
```

### Вариант 3: Screen/tmux (простой способ)

```bash
# С использованием screen
screen -S shop-q
python main.py
# Нажмите Ctrl+A, затем D для отсоединения

# Вернуться к сессии
screen -r shop-q

# С использованием tmux
tmux new -s shop-q
python main.py
# Нажмите Ctrl+B, затем D для отсоединения

# Вернуться к сессии
tmux attach -t shop-q
```

### Вариант 4: Supervisor

Установка Supervisor:

```bash
sudo apt-get install supervisor
```

Создайте файл `/etc/supervisor/conf.d/shop-q.conf`:

```ini
[program:shop-q]
command=/usr/bin/python3 /path/to/shop-q/main.py
directory=/path/to/shop-q
user=your_username
autostart=true
autorestart=true
stderr_logfile=/var/log/shop-q.err.log
stdout_logfile=/var/log/shop-q.out.log
environment=BOT_TOKEN="сюда токен бота",NOTIFICATION_CHANNEL_ID="сюда ID группы для уведомлений"
```

**Управление:**

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start shop-q
sudo supervisorctl status shop-q
```

### Очистка логов

**Просмотр размера логов:**
```bash
ls -lh shop_q.log
```

**Очистка файла логов:**
```bash
# Полная очистка
> shop_q.log

# Или удаление старых логов
rm shop_q.log
```

**Ротация логов (рекомендуется):**

Создайте файл `/etc/logrotate.d/shop-q`:
```
/path/to/shop-q/shop_q.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 your_username your_username
    postrotate
        systemctl reload shop-q > /dev/null 2>&1 || true
    endscript
}
```

### Очистка кеша базы данных

**Резервное копирование перед очисткой:**
```bash
# Создать бэкап
cp shop_q.db shop_q_backup_$(date +%Y%m%d_%H%M%S).db
```

**Оптимизация базы данных:**
```bash
# Запустите VACUUM для очистки и оптимизации
sqlite3 shop_q.db "VACUUM;"

# Проверка размера БД
ls -lh shop_q.db
```

**Очистка старых данных (опционально):**
```bash
# Подключитесь к БД
sqlite3 shop_q.db

# Удалите старые заказы (например, старше 90 дней)
DELETE FROM orders WHERE created_at < datetime('now', '-90 days');

# Удалите старые корзины неактивных пользователей
DELETE FROM cart WHERE added_at < datetime('now', '-30 days');

# Оптимизируйте после удаления
VACUUM;

# Выход
.quit
```

# 2. Сделайте бэкап

# 3. Получите обновления

### Диагностика проблем

**Бот не отвечает:**
1. Проверьте, запущен ли процесс: `ps aux | grep python`
2. Проверьте логи: `tail -50 shop_q.log`
3. Проверьте токен: убедитесь, что BOT_TOKEN установлен
4. Проверьте интернет-соединение

**Ошибки базы данных:**
```bash
# Проверка целостности БД
sqlite3 shop_q.db "PRAGMA integrity_check;"

# Если БД повреждена, восстановите из бэкапа
cp shop_q_backup.db shop_q.db
```

**Проблемы с памятью:**
```bash
# Очистите кеш системы
sync; echo 3 > /proc/sys/vm/drop_caches

# Перезапустите бота
sudo systemctl restart shop-q
```

## 💾 База данных

Бот использует **SQLite** базу данных (`shop_q.db`).

### Структура таблиц:

1. **users** - Пользователи
   - user_id, username, first_name, phone
   - role (pending/buyer/manager/admin)
   - balance, blocked, verification_status
   - referrer_id, referral_code

2. **countries** - Страны
   - id, name, created_at, created_by

3. **cities** - Города
   - id, country_id, name, created_at, created_by

4. **districts** - Районы
   - id, city_id, name, created_at, created_by

5. **categories** - Категории товаров
   - id, name, parent_id (для подкатегорий)

6. **products** - Товары
   - id, seller_id, name, description, price, stock
   - category_id, country_id, city_id, district_id
   - image_path, status

7. **orders** - Заказы
   - id, order_number, buyer_id, seller_id, product_id
   - quantity, total_price, status, payment_method

8. **cart** - Корзина
   - user_id, product_id, quantity

9. **ratings** - Рейтинги и отзывы
   - order_id, buyer_id, seller_id, product_id
   - rating (1.1-5.5), comment

10. **favorites** - Избранное
    - user_id, product_id

11. **verification_requests** - Заявки на верификацию
    - user_id, status, requested_at, processed_by

12. **transactions** - Транзакции
    - user_id, amount, type, description

13. **promo_codes** - Промо-коды
    - code, discount_percent, usage_limit

14. **support_tickets** - Тикеты поддержки
    - user_id, subject, message, status

15. **complaints** - Жалобы
    - complainant_id, target_id, product_id, reason

### Резервное копирование:

```bash
# Создать копию базы данных
cp shop_q.db shop_q_backup_$(date +%Y%m%d).db

# Автоматическое резервное копирование (cron)
0 2 * * * cp /path/to/shop_q.db /path/to/backups/shop_q_$(date +\%Y\%m\%d).db
```
